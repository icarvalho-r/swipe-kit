# FASE 3 — HOOKS

## Objetivo
Gerar a MENSAGEM bruta de cada hook. Sem formato, sem pensar em vídeo. Só a ideia que para o scroll.

## Etapa 3.1 — Geração

Gerar **60 hooks** (12 por combinação de ângulos). Cada hook é frase ou micro-parágrafo.

**Tom:** seguir padrões da Fase 1.5. Consultar calibrador de tom.

### Regra de Checagem Cruzada (OBRIGATÓRIA)

Antes de apresentar, validar CADA hook contra o Glossário do Avatar (Fase 1):
- Se um termo NÃO aparece na pesquisa de público NEM nos ads validados → substituir por equivalente do glossário OU sinalizar como "fora do vocabulário" com justificativa.
- Termos exclusivos da VSL NÃO devem ser usados nos hooks, a menos que também apareçam nos ads validados.

## Etapa 3.2 — Filtragem

Filtrar os **20 melhores hooks**.

### Critérios (em ordem de prioridade):

1. **Loop aberto** — Cria pergunta/tensão que PRECISA ser resolvida? Se a frase se basta sozinha, é fraca.
2. **Especificidade > Generalidade** — Números, situações concretas, detalhes palpáveis.
3. **Tensão emocional** — Provoca algo visceral (medo, raiva, desejo, indignação, esperança)?
4. **Linguagem do avatar** — Usa palavras que o avatar diria? Checar contra Glossário.
5. **Quebra de padrão** — Surpreende? Contraria expectativa?

### Formato de apresentação:

```
[nº] HOOK: "texto do hook"
    COMBO: [Ângulo A + Ângulo B]
    POR QUE PASSOU: [1 frase com base nos critérios]
```

## Etapa 3.3 — Edição Colaborativa

Fluxo COLABORATIVO após apresentar os 20:
- Usuário seleciona, edita, reescreve, devolve os que quer manter.
- Pode devolver com lacunas (hooks faltando).
- Claude completa lacunas sugerindo opções.
- **NÃO avançar até o usuário fechar a lista completa de 20 hooks.**

**Aguardar lista fechada antes de avançar para Fase 4.**
