# FASE 1.5 — ANÁLISE DE ESTILO DOS ADS VALIDADOS

## Objetivo
Extrair padrões de estilo dos ads que já performaram. A VSL é fonte de CONTEÚDO. Os ads validados são fonte de ESTILO.

## O que Analisar

1. **Tom predominante** — Formal? Educacional? Conversa de amigo? Desabafo? Autoridade?
2. **Estrutura recorrente** — Como abrem (identificação? pergunta? afirmação?)? Como desenvolvem (espelham dor? explicam mecanismo?)? Como fecham (CTA direto? elaborado?)?
3. **Vocabulário-chave** — Palavras/expressões que se repetem nos ads validados
4. **Proporção dor vs. mecanismo** — % do body que é espelhamento da experiência do avatar vs. explicação técnica
5. **Estilo de CTA** — Curto e direto? Longo e elaborado? Com escassez? Com exclusividade?

## Formato de Apresentação

```
PADRÕES DE ESTILO IDENTIFICADOS:

TOM: [descrição]
ESTRUTURA: [como abrem → desenvolvem → fecham]
VOCABULÁRIO RECORRENTE: [lista de termos]
PROPORÇÃO DOR vs. MECANISMO: [ex: "70% espelhamento de dor, 30% mecanismo"]
ESTILO DE CTA: [descrição]
```

## Calibrador de Tom

Criar 2-3 FRASES-EXEMPLO no tom identificado. Essas frases são referência interna — NÃO serão usadas nos scripts. Servem para o Claude consultar antes de escrever hooks, bodys e CTAs.

```
CALIBRADOR DE TOM:
Frase 1: "[exemplo]"
Frase 2: "[exemplo]"
Frase 3: "[exemplo]"
```

## Regra de Tom (aplica em TODAS as fases seguintes)

O tom dos scripts ESPELHA o tom dos ads validados. Se coloquiais → coloquiais. Se formais → formais. Manter consistência em hooks, bodys e CTAs.
