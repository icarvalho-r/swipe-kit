## QG-FINAL: Quality Gate — VSL Completa

### 1. As 3 Vendas

| # | Check | Severidade | Status |
|---|-------|-----------|--------|
| 1.1 | Lead vende o CONTEUDO (nao o produto) | BLOCKING | |
| 1.2 | Historias + Tese vendem a ESPERANCA | BLOCKING | |
| 1.3 | PBU + Oferta + Close vendem o VALOR | BLOCKING | |

### 2. Tese de Marketing

| # | Check | Severidade | Status |
|---|-------|-----------|--------|
| 2.1 | Argumento 1 (Causa) tem: Ponto de Partida + Peca Faltante + Crenca Comum + Grande Pesquisa | BLOCKING | |
| 2.2 | Argumento 2 (Funcao) tem: Conclusao do Estudo + pelo menos 2 tipos de Grande Prova | BLOCKING | |
| 2.3 | Argumento 3 (Solucao) tem: Pergunta Paradoxal + Explicacao + Evidencias + Teste + Grande Resultado | BLOCKING | |
| 2.4 | A logica progride sequencialmente: causa → prova → solucao | BLOCKING | |
| 2.5 | A linguagem e acessivel (nao academica) | WARNING | |
| 2.6 | Nao menciona produto na tese (apenas mecanismo) | BLOCKING | |

### 3. Lead

|---|-------|-----------|--------|
| 3.1 | Lead NAO menciona produto ou venda | BLOCKING | |

| # | Check | Severidade | Status |

---

## Criterios de Aprovacao

- **PASS:** Todos os checks BLOCKING passam + pelo menos 80% dos WARNING passam
- **NEEDS_REVISION:** 1-2 checks BLOCKING falham → corrigir e revalidar
- **FAIL:** 3+ checks BLOCKING falham → revisao profunda necessaria

## VETO Conditions (auto-reprovacao)

1. Lead que menciona produto ou venda → VETO
2. Tese sem os 3 argumentos completos → VETO
3. Mecanismo inventado (sem historico de mercado) → VETO
4. Oferta sem garantia → VETO
